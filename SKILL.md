# AI成本优化分析器

## 触发条件
当用户要求分析账单、优化成本、找浪费点时激活。

## 输入要求
- API账单日志文件（JSON/CSV格式）
- 或提供账单时间段（自动从API拉取）

## 分析维度

### 1. Token浪费检测
- **重复调用识别**：相同或高度相似的prompt在短时间内重复调用
- **过长上下文**：每次调用携带的历史消息超过实际需要
- **无效工具调用**：调用后未使用结果的工具请求
- **截断损失**：因max_tokens设置过低导致的截断重试

### 2. 成本异常检测
- 单次调用成本超过平均值3个标准差
- 特定时间段成本激增（如深夜批量任务）
- 特定工具/函数的成本占比异常

### 3. 优化建议输出
- 可节省金额估算
- 具体代码修改建议
- 配置优化方案（缓存策略、max_tokens调整）

## 输出格式

```markdown
# 成本分析报告

## 概览
- 总调用次数：X
- 总Token消耗：X
- 总成本：$X
- **可节省：$X (X%)**

## 浪费点Top 3
1. [类型] 描述 | 浪费金额 | 优化方案
2. ...
3. ...

## 具体优化清单
- [ ] 启用Redis缓存，减少重复调用，预计节省$X
- [ ] 裁剪system prompt，从X token降至Y token
- [ ] 设置工具调用超时，避免无效等待
```

## 使用示例
```
用户：分析我这周的API账单，看看钱花哪了
[上传账单文件]

助手：[输出成本分析报告]
```
